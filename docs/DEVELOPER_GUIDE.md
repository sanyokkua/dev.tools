# dev.tools — Developer Guide

> **Version:** 3.0.0 | **Framework:** Next.js 16 (Pages Router) | **Language:** TypeScript + React 19
> **Deployment:** Static export → GitHub Pages | **Live:** [sanyokkua.github.io/dev.tools](https://sanyokkua.github.io/dev.tools/)

---

## What this is

Browser-based developer utilities suite — 24 tools spanning text manipulation, code editing,
hashing, encoding, JWT/Cron/QR generation, software installer scripting, and AI tooling.
Fully client-side: nothing leaves the browser. Deployed as a static export to GitHub Pages.

---

## Quick start

```bash
npm install
npm run dev        # localhost:3000
```

---

## Commands

| Command                 | Purpose                                                                                   |
| ----------------------- | ----------------------------------------------------------------------------------------- |
| `npm run dev`           | Dev server on port 3000                                                                   |
| `npm run build`         | Production build (static export to `./out`)                                               |
| `npm run validate:sw`   | Verify `public/sw.js` precache covers all routes (run after every build)                  |
| `npm run lint`          | Run ESLint                                                                                |
| `npm run lint:fix`      | Run ESLint with auto-fix                                                                  |
| `npm run test`          | Jest with coverage                                                                        |
| `npm run format`        | Format with Prettier                                                                      |
| `npm run check:format`  | Check formatting without writing                                                          |
| `npm run verify`        | Full pipeline: format → lint → test                                                       |
| `npm run clean`         | Remove build artifacts                                                                    |
| `npm run verify:ui`     | Static pass (routes × 3 widths × 2 themes) + interaction smoke tests                      |
| `npm run verify:smoke`  | Interaction smoke tests only                                                              |
| `npm run build:prompts` | Validate TypeScript catalog + regenerate `manifest.generated.ts` / `loaders.generated.ts` |

Run a single test file:

```bash
npx jest test/path/to/file.test.ts
```

```
src/

│   └── layouts/        Flex/grid container primitives

└── styles/             40+ SCSS/CSS files — all globally imported in _app.tsx
```

See [ARCHITECTURE.md](ARCHITECTURE.md) for full details including Mermaid diagrams.

## Context providers

Five React Context providers wrap every page (see `src/pages/_app.tsx`):

| Provider                 | Hook                  | Purpose                       |
| ------------------------ | --------------------- | ----------------------------- |
| `ThemeProvider`          | `useTheme()`          | light/dark theme toggle       |
| `PageProvider`           | `usePage()`           | page title + help-panel state |
| `ToasterProvider`        | `useToast()`          | toast notifications           |
| `FileOpenProvider`       | `useFileOpen()`       | file picker dialog            |
| `FileSaveDialogProvider` | `useFileSaveDialog()` | file save dialog              |

All live in `src/components/contexts/`.

## Page patterns

Three patterns cover all 24 routes:

| Pattern      | Example page          | Description                                                                     |
| ------------ | --------------------- | ------------------------------------------------------------------------------- |
| **ToolView** | `/string-utils`       | Left input + middle function list + right output. Logic in `utils-factory.ts`.  |
| **Editor**   | `/code-editor`        | Split pane: Monaco editor + preview. Uses `.editorpane/.eh/.eb` CSS primitives. |
| **Custom**   | `/software-installer` | Bespoke layout for complex tool UX.                                             |

See [howto/add-a-tool-page.md](howto/add-a-tool-page.md) for step-by-step instructions.

## Testing

```bash
npm run verify          # format + lint + unit/coverage
npm run verify:ui       # Playwright: all routes × 3 widths × 2 themes
npm run verify:smoke    # Interaction smoke tests (28+ flows)
```

Screenshots save to `.tmp/verify-screens/`. The UI verify gate fails on:

- Horizontal overflow (`scrollWidth > clientWidth + 1px`)
- Console errors / page errors
- Serif font detected on `document.body`
- Monaco editor height ≤ 200px on editor pages

Tests live in `test/`. Config: `jest.config.ts` (jsdom environment).

## Static export + PWA

`next.config.mjs` sets `output: 'export'`. `basePath` and `assetPrefix` are set dynamically
from `GITHUB_REPOSITORY` env var during CI. In development these are empty strings.

PWA via `@ducanh2912/next-pwa` + Workbox — production-only; `public/sw.js` precaches all routes
with an 8 MB file size limit. Run `npm run validate:sw` after every build.

---

## CI/CD

GitHub Actions (`.github/workflows/nextjs.yml`) — three jobs:

1. **test** — format check, lint, jest
2. **build** — `next build` → static `./out` artifact
3. **deploy** — GitHub Pages publish

Triggers on push to `master`.
